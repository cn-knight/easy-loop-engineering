Coding Plan 个人版

套餐概览

方舟 Coding Plan 是为开发者量身定制的订阅服务，助力高效编码开发。具备如下优势：

- 支持主流模型 ：不仅支持主流大语言模型，还支持 Embedding 向量化模型，涵盖 Doubao Seed Code、Doubao Seed 2.0 系列、GLM 系列、MiniMax 系列、Kimi系列、DeepSeek 系列、Doubao Embedding Vision 等模型。

- 兼容主流 AI 编码工具 ：支持 Claude Code、OpenCode、OpenClaw、TRAE、Cline、Cursor、Roo Code、Kilo Code 等主流编程工具，套餐额度共享，适配多种开发场景。

- 多种套餐适配不同场景 ：提供 Lite 与 Pro 套餐，适应普通与高强度编程场景。

- 稳定平稳运行 ：平台具有多租户隔离能力，调用高峰不会明显降速，TPM 满足正常开发需求。

说明

- 已了解方舟 Coding Plan 的用户，可以访问 [方舟 Coding Plan 活动](https://www.volcengine.com/activity/codingplan) 按需订阅套餐，参见 [快速开始](https://www.volcengine.com/docs/82379/1928261?lang=zh) 开启 AI 编程之旅。

套餐详情

用量预估说明

注意

方舟已推出 Agent Plan 套餐，新增支持全模态模型及专属 Harness，采用精细化 AFP 抵扣规则，用量清晰可查。如需使用，请前往 [方舟 Agent Plan 活动](https://ai.volcengine.com/activity/agentplan?_vtm_=a441938.b656048.0_0.0_0.0.2709_7612576610202109478) 订阅套餐。

资源消耗因请求上下文长度和模型差异巨大，实际可用次数会因此有较大波动。

例如 OpenClaw、Claude Code 开启 Agent Team 模式资源消耗会显著增加，建议仅在复杂多任务开发场景下使用，避免额度快速消耗。

<iframe src="//about:blank" frameborder="0"></iframe>

| 套餐 | 适用场景 |
| --- | --- |
| Lite 套餐 | 中等强度的开发任务，适合大多数开发者。 |
| Pro 套餐 | 复杂项目开发，适合高强度工作的开发者。 |

实际可用量会因项目复杂度、代码库规模、是否启用自动接受、是否开启 thinking 等因素而有所不同。套餐用量信息可在 [开通管理页面](https://console.volcengine.com/ark/region:ark+cn-beijing/openManagement?LLM=%7B%7D&advancedActiveKey=subscribe) 查看。

套餐有效期

套餐有效期以自然月为单位，自购买成功当日开始计算。1 个月套餐有效期示例如下：

<iframe src="//about:blank" frameborder="0"></iframe>

| 购买日期 | 购买时长 | 结束日期 |
| --- | --- | --- |
| 01.04 | 1 个月 | 02.04 23:59 |
| 01.31 | 1 个月 | 02.28 23:59 |
| 02.01 | 1 个月 | 03.01 23:59 |

套餐额度刷新规则

套餐额度在时间周期内耗尽后，需要等待下一个周期自动恢复额度，不会消耗其他资源包或账户余额。

- 每5小时限额：根据首次请求发生时间，以 5 小时为周期定时刷新限额。

- 周限额：每周一 00:00:00 重置周限额。

- 月限额：每订阅月第 1 日 00:00:00 重置月限额。

套餐额度使用限制

- 同一订阅套餐可在所有支持的工具中使用，额度共享。

- 套餐额度仅在 AI 编程工具中生效，不可用于 API 调用。在非 AI 编程工具中使用方舟 Coding Plan 权益对应的 Base URL 和 API Key 有可能被识别为滥用/违规，会导致订阅停用或账号封禁。

接入编程工具

注意

务必使用 Coding Plan 支持的模型及 Base URL：https://ark.cn-beijing.volces.com/api/coding/v3（兼容 OpenAI 协议）或https://ark.cn-beijing.volces.com/api/coding（兼容 Anthropic 接口协议）。

- 其他模型无法在 Coding Plan 中使用。

- 如未使用指定的 Base URL，将无法使用 Coding Plan 额度，并可能产生额外 API 请求的费用。

支持的模型

支持主流 Code 模型，不同区域可选择的模型请以控制台为准。模型配置支持以下两种方式（具体参见 [模型配置](https://www.volcengine.com/docs/82379/1928261?lang=zh#ngCxjYAzHr) ）：

- 配置 Model Name，实时切换模型。

- 配置 ark-code-latest，在控制台切换模型。

说明

在控制台可以查看模型在编码工具中默认的 thinking 行为，真实使用时可由编码工具通过指定参数修改思考行为。

注意

glm-5.2、deepseek-v4-flash、deepseek-v4-pro 支持 1M 上下文窗口用于包含大型代码库的长会话。在工具中开启 1M 上下文的方案可参见 [Claude Code](https://www.volcengine.com/docs/82379/1928262?lang=zh#7a3e5c1b) 、 [OpenCode](https://www.volcengine.com/docs/82379/2188958?lang=zh#baa2dc81) 、 [OpenClaw](https://www.volcengine.com/docs/82379/2183190?lang=zh#96962158) 。
